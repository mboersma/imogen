# Contributing

This project welcomes contributions and suggestions. Most contributions require you to agree to a
Contributor License Agreement (CLA) declaring that you have the right to, and actually do, grant us
the rights to use your contribution. For details, visit https://cla.opensource.microsoft.com.

When you submit a pull request, a CLA bot will automatically determine whether you need to provide a
CLA and decorate the PR appropriately, with a status check or comment. Just follow the instructions
from the bot. You only need to do this once across all repos using our CLA.

This project has adopted the [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/).
For more information see the [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/)
or contact [opencode@microsoft.com](mailto:opencode@microsoft.com) with any questions or comments.

## Working in this repo

Read [AGENTS.md](AGENTS.md) before you start. It covers the architecture, conventions, and how the
agent and its tools are structured. Keep `AGENTS.md`, `README.md`, and `docs/plan.md` up to date in
the same change when your work affects them.

Keep commit messages brief and code comments terse. Don't commit secrets; all Azure auth uses
Workload Identity.
